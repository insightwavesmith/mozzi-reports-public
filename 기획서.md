# 모찌 리포트 (Mozzi Reports) 기획서

> 2026-02-15 | 에이전트팀 리뷰 보고서를 웹에서 열람하는 보고용 서비스

## 배경
- Smith님은 Mac Studio(모니터 없음) + 슬랙으로 업무
- 에이전트팀 리뷰 보고서가 로컬 HTML 파일이라 확인 불가
- 슬랙에서 링크 하나로 보고서를 바로 열람할 수 있어야 함

## 핵심 컨셉
Vercel 프로젝트 **1개**로 모든 보고서를 호스팅하는 정적 사이트.
프레임워크 없이 순수 HTML/CSS. 보고서 HTML 파일을 넣고 push하면 끝.

## URL 구조
```
mozzi-reports.vercel.app/                         → 보고서 목록
mozzi-reports.vercel.app/reports/2026-02-15-xxx    → 개별 보고서
```

## 페이지

### 1. 인덱스 (목록)
- 카드 리스트: 날짜 / 제목 / 프로젝트명 / 상태 뱃지
- 상태: 🟡 리뷰중 → 🟠 승인대기 → 🟢 완료 → 🔴 반려
- 최신순 정렬
- 모바일 반응형 (슬랙 인앱 브라우저 대응)
- Pretendard 폰트 + 자사몰사관학교 브랜드 컬러

### 2. 보고서 상세
- 기존 HTML 보고서를 iframe 또는 직접 서빙
- 상단 네비: ← 목록으로 | 프로젝트명 | 날짜 | 상태
- 하단: 승인/반려 상태 표시 (추후 슬랙 연동 가능)

## 파일 구조
```
mozzi-reports/
├── public/
│   └── reports/
│       ├── 2026-02-15-email-logo-greeting.html
│       └── ...
├── scripts/
│   └── build-index.js    ← reports/ 스캔 → index.html 자동 생성
├── index.html             ← 빌드 결과물
├── vercel.json
├── package.json
└── 기획서.md
```

## 기술 스택
- **순수 HTML/CSS** (TailwindCSS CDN)
- **Node.js 스크립트** 1개 (build-index.js): reports/ 폴더 스캔 → 메타데이터 추출 → index.html 생성
- **Vercel Static** 배포 (빌드 커맨드: `node scripts/build-index.js`)
- 프레임워크 없음, 의존성 0개 (devDependencies만)

## 메타데이터 규칙
보고서 HTML의 `<title>` + `<meta>` 태그에서 자동 추출:
```html
<title>이메일 로고/인사말 리뷰 보고서 — 2026-02-15</title>
<meta name="project" content="qa-helpdesk">
<meta name="status" content="pending">   <!-- pending | approved | rejected -->
<meta name="date" content="2026-02-15">
```

## 워크플로우
1. 에이전트팀 → `docs/review/xxx.html` 생성 (기존과 동일)
2. 모찌 → `mozzi-reports/public/reports/` 에 복사
3. `git push` → Vercel 자동 배포 (빌드 시 index 재생성)
4. 슬랙에 링크 공유: `mozzi-reports.vercel.app/reports/xxx.html`
5. Smith님 슬랙에서 바로 확인

## 슬랙 연동 (OG 태그)
```html
<meta property="og:title" content="리뷰 보고서: 이메일 로고/인사말">
<meta property="og:description" content="T1. 로고 v4 교체 / T2. 인사말 삭제">
<meta property="og:image" content="...브랜드 로고...">
```
→ 슬랙에서 링크 붙여넣기하면 프리뷰 카드 자동 생성
