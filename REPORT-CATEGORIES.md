# 모찌리포트 카테고리 규칙

> 리포트 생성 시 반드시 이 규칙에 맞는 디렉토리에 파일을 넣어라.

## 카테고리 목록

| 카테고리 | 디렉토리 | 용도 | 예시 |
|---------|---------|------|------|
| 기획서 | `plan/` | 서비스 기획, 설계서, 마스터 아키텍처, 프레임워크 | 마스터 설계서, 소재 분석 기획, 임베딩 설계 |
| 아키텍처 | `architecture/` | 기술 아키텍처, 시스템 구조도, 파이프라인 설계 | 데이터 수집 구조, RAG 아키텍처, 파이프라인 |
| 목업 | `mockup/` | UI/UX 목업, 디자인 시안, 프로토타입 | 총가치각도기 목업, 소재 분석 탭 목업 |
| 리서치 | `research/` | 외부 조사, 논문 분석, 경쟁사 조사, 기술 리서치 | SEO 분석, 뇌 영감 아키텍처, Motion 가이드 |
| 마케팅 | `marketing/` | 마케팅 전략, 로드맵, 키워드 분석 | 오가닉 마스터 플랜, Q2 로드맵 |
| 코드 리뷰 | `review/` | SDK/에이전트팀 코드 리뷰, 품질 체크 | TASK 코드리뷰, 보안 리뷰 |
| 결과 보고 | `release/` | 배포 완료 보고, 릴리스 노트 | 파이프라인 릴리스, 기능 배포 보고 |
| 태스크 | `task/` | TASK 정의서, 개발 태스크 분해 | TASK 정의서, 체크리스트 |
| QA | `qa/` | QA 체크리스트, 성능 리뷰, 테스트 결과 | 전체 QA 체크리스트, 성능 코드리뷰 |
| 보안 | `security/` | 보안 감사, 보안 수정 리뷰 | 보안 코드리뷰, 보안 감사 |
| 운영 | `ops/` | 크론 현황, 운영 보고, 인프라 상태 | 크론 대시보드, 개발 보고서 |
| 분석 | `analysis/` | 데이터 분석 결과, 수치 분석 | 타겟중복률 분석 |
| 아카이브 | `archive/` | 더 이상 활성 아닌 과거 리포트 | 뉴스레터 시절 리포트 |

## 파일명 규칙

```
{YYYY-MM-DD}-{영문-케밥-케이스}.html
```

예시:
- `2026-03-23-andromeda-feed-integration.html`
- `2026-03-22-creative-deep-analysis.html`

날짜 없는 파일은 `N/A`로 정렬됨 → **반드시 날짜 붙일 것**.

## 판단 기준 (애매할 때)

1. **"뭘 만들지" 결정하는 문서** → `plan/`
2. **"어떻게 만들지" 기술 구조** → `architecture/`
3. **"어떻게 보일지" 화면** → `mockup/`
4. **"외부 조사/분석"** → `research/`
5. **"코드 품질 확인"** → `review/`
6. **"만들었다, 결과"** → `release/`
7. **"이걸 만들어라" 지시서** → `task/`

## 에이전트팀 규칙

- 리포트 생성 시 이 파일(`REPORT-CATEGORIES.md`)을 먼저 읽고 카테고리 판단
- 애매하면 `plan/`에 넣고 나중에 모찌가 분류
- `archive/`는 직접 넣지 말 것 — 모찌가 판단해서 이동
- 커밋 메시지: `리포트: {카테고리} - {제목}` 형식
