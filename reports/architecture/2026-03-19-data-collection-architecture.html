🔍 collect-daily 상세 플로우

왜 중요한가? 하루의 모든 것이 여기서 시작됩니다. 이 크론이 실패하면 그날의 광고 분석이 없어요. 수집 → 저장 → 분석 파이프라인 호출까지 한 번에 처리합니다.
  1. ad_accounts 테이블에서 활성 계정 목록 조회
  2. 계정별 Meta Marketing API 호출 — ad 기본정보 + insight 15개 지표 한 번에 수집
  3. daily_ad_insights에 UPSERT (날짜+광고ID 기준, 중복 없이 갱신)
  4. creative 정보 → ad_creative_embeddings에 UPSERT (media_url, ad_copy, lp_url 등)
  5. Railway /pipeline POST → L1(태깅) → L2(시선) → L3(벤치마크) → L4(점수) 순차 실행
  6. media_url 3단계 fallback: image_url → video_id 썸네일 → image_hash → null

🔍 collect-benchmarks 상세 플로우

비유: 이건 주간 품질 감사입니다. 매주 화요일 새벽에 전체 공장 성과를 다시 측정해서 업계 기준값(벤치마크)을 업데이트합니다. 수강생 소재가 "평균 대비 어느 수준인지" 판단하는 기준이 돼요.
  1. STEP 0 — 계정 카테고리 분류: profiles 매칭 → AI 자동 분류
  2. STEP 1 — Meta API 원본 수집 → ad_insights_classified UPSERT
  3. STEP 2 — creative_type × ranking_type × ranking_group × category별 Trimmed Weighted Mean → benchmarks UPSERT
  4. STEP 3 — MEDIAN_ALL (랭킹 무관 전체 중앙값) 추가
수치로 보면:
13개 지표 × 3 ranking_type (engagement / conversion / quality) × 3 ranking_group (above_avg / average / below_average) = 총 117개 벤치마크 포인트

측정 지표: video_p3s_rate, thruplay_rate, retention_rate, reactions_per_10k, comments_per_10k, shares_per_10k, saves_per_10k, engagement_per_10k, ctr, click_to_checkout_rate, click_to_purchase_rate, checkout_to_purchase_rate, roas

📣 광고 소재 분석 — 어떻게 하나?

핵심 아이디어: "잘 되는 광고"와 "내 광고"를 비교해서, 왜 차이가 나는지를 알려주는 것.
숫자만 보여주는 게 아니라 → "이 부분이 부족하니까 이렇게 고쳐"까지.

현재 분석 방법 (AS-IS)

1단계: 데이터 수집

매일 새벽 3시 — collect-daily 크론
수강생이 연결한 광고계정에서 Meta Marketing API로 전 광고 성과 자동 수집

수집 항목 (15개 핵심 지표):
• 비용: spend, impressions, clicks
• 기반: video_p3s_rate(3초 시청률), thruplay_rate(완시청률), retention_rate(잔존율)
• 참여: reactions, comments, shares, saves(만 노출 당 수치)
• 전환: ctr(클릭률), click_to_checkout_rate(결제시작률), checkout_to_purchase_rate(결제→구매율), roas(광고수익률)
• 랭킹: quality_ranking, engagement_ranking, conversion_ranking (Meta AI 자동 등급)

2단계: 벤치마크 수집 (비교 기준)

매주 화요일 새벽 2시 — collect-benchmarks 크론
"잘 하는 광고계정들"의 데이터를 따로 수집해서 기준값을 만듦

어떻게 계산하나?
1. 전체 광고계정의 데이터를 Meta API로 재수집
2. 동영상/이미지 × 품질등급(상/중/하) × 카테고리별로 나눔
3. Trimmed Weighted Mean(이상치 제거 가중평균)으로 13개 지표의 기준값 산출
4. 총 117개 벤치마크 포인트 생성

3단계 구조 (총가치각도기):
🟢 기반: 3초시청률, 완시청률, 잔존율, CTR → "사람들이 보긴 하나?"
🟡 참여: 반응, 댓글, 공유, 저장 → "관심을 갖나?"
🔴 전환: 결제시작률, 구매전환율, ROAS → "돈이 되나?"

3단계: AI 소재 분석 (L1~L4)

collect-daily 완료 후 자동 실행 — Railway /pipeline

L1 태깅 (Gemini 2.5 Pro Vision):
소재 이미지를 보고 12개 요소를 자동 추출
→ hook 유형(문제제기/질문/혜택/충격/없음), 스타일, CTA 유형/위치, 색감, 사람 유무

L2 시선 예측 (DeepGaze IIE):
"사람이 이 소재를 볼 때 어디를 먼저 보는가" 히트맵 생성

L3 요소별 벤치마크:
L1에서 뽑은 요소 × 실제 성과를 연결 → "problem hook 소재의 평균 ROAS는 2.35"

L4 종합 점수 + 개선 제안 (Gemini 2.5 Pro):
소재 이미지 + L1 태그 + L3 벤치마크를 종합해서 5개 영역 점수 + 구체적 제안
→ visual_impact, message_clarity, cta_effectiveness, social_proof, lp_consistency

4단계: 수강생에게 보여주기

총가치각도기 → 소재 분석 탭

현재:
• 벤치마크 대비 "내 CTR은 상위 30%" 같은 숫자 비교
• L4 종합 점수 + 개선 제안 텍스트

추가 필요 (TO-BE):
• 소재 이미지/동영상 직접 보여주기 + 클릭 시 상세 분석
• 3단계(기반/참여/전환) 각각에서 뭐가 부족한지 시각화
• "이 hook 유형은 전환에 약하다 → 이런 hook으로 바꿔봐" 같은 구체 제안
• 잘 되는 소재의 공통 패턴 요약 ("상위 20% 소재는 이런 특징이 있다")

🔮 발전 방향 (Smith님 구상)

벤치마크 계정의 "소재 자체"도 수집해서 분석하면?

지금은 벤치마크 = 숫자(지표)만 수집하는데,
벤치마크 계정의 광고 소재 이미지/동영상 + LP까지 수집하면:

1. 잘 되는 소재 100개 → L1 태깅 → "성과 좋은 소재의 공통 패턴" 추출
   예: "상위 20% 소재의 70%는 problem hook + 사람 등장 + 하단 고정 CTA"

2. 수강생 소재와 패턴 비교 → "왜 네 소재가 안 되는지" 근거 있는 진단
   예: "너는 hook이 없고, CTA가 안 보이고, 사람이 안 나와 — 이 3개가 성과 차이 원인"

3. 숫자 비교("48% 부족") → 원인 진단 + 액션 제안으로 진화

🖥️ LP(상세페이지) 분석 — 어떻게 하나?

핵심 아이디어: 광고가 좋아도 LP가 별로면 구매 안 됨.
"광고 클릭 → LP 도착 → 구매"까지의 흐름을 끊김 없이 만들어야 해요.
그래서 소재↔LP 일관성이 중요합니다.

현재 LP 분석 플로우 (AS-IS)

1단계: LP URL 추출

collect-daily에서 자동 추출
광고 데이터 수집할 때 각 소재의 lp_url(랜딩페이지 주소)도 함께 저장됩니다.
→ ad_creative_embeddings 테이블의 lp_url 컬럼

2단계: LP 크롤링 (스크린샷 수집)

매시간 — crawl-lps 크론
Railway의 Puppeteer 서버가 LP에 접속해서:

1. 풀페이지 스크린샷 캡처 (JPEG, 모바일 뷰포트)
2. CTA 버튼 클릭 후 스크린샷 — "구매하기" 버튼 누른 후 화면도 캡처
3. Supabase Storage에 이미지 업로드
4. 스크린샷 해시로 변경 감지 — 같은 LP가 바뀌었는지 자동 감지

현재 속도: 매시간 20건 → 하루 ~480건 처리
현재 진행: 1,522건 큐 중 942건 pending (약 2일 소요)

3단계: LP 구조 분석 (AI Vision)

Gemini 2.0 Flash Vision — scripts/analyze-lps.mjs
크롤링된 스크린샷을 AI가 보고 LP의 구조를 자동 분석:

• Hero 섹션: 메인 이미지/영상, 헤드라인 텍스트
• CTA 버튼: 위치, 색상, 문구, 개수
• 가격/혜택: 할인율, 가격 표시 방법
• 리뷰/후기: 있는지, 몇 개, 형태
• FAQ: 있는지, 어떤 질문들
• 전체 구조: 섹션 순서, 길이, 모바일 최적화 여부

→ lp_structure_analysis 테이블에 저장 (현재 90건 분석 완료)

4단계: 소재↔LP 일관성 점수

Gemini 임베딩 3072차원 + 코사인 유사도

광고 소재와 LP가 얼마나 일관성 있는지 자동 측정:

1. 소재 이미지/텍스트 → 3072차원 벡터 (이미 ad_creative_embeddings에 있음)
2. LP 스크린샷/텍스트/CTA → 3072차원 벡터
3. 두 벡터의 코사인 유사도 계산:
  • visual_similarity — 소재 이미지 vs LP 이미지
  • text_similarity — 소재 카피 vs LP 텍스트
  • cross_vt — 소재 이미지 vs LP 텍스트
  • cross_tv — 소재 텍스트 vs LP 이미지
  • total_score — 가중 평균

→ creative_lp_consistency 테이블에 저장 (현재 170건 분석 완료)

🔮 발전 방향 (Smith님 구상)

벤치마크 계정의 LP도 수집해서 비교하면?

지금 벤치마크 = 전환율 숫자만 알고 LP 자체는 안 봄.
전환율 상위 계정의 LP를 크롤링 + 구조 분석하면:

1. "잘 되는 LP의 공통 패턴" 추출
  예: "전환율 상위 LP의 90%는 리뷰 섹션 있음, 80%는 CTA 3개 이상, 75%는 Hero에 사람 사진"

2. 수강생 LP와 패턴 비교 → "왜 네 LP에서 이탈하는지" 진단
  예: "리뷰 섹션 없음 / CTA가 1개뿐 / 가격이 스크롤 3번 해야 보임"

3. LP 점수 산출 → 광고 소재 점수와 함께 통합 인사이트
  예: "소재는 80점인데 LP가 45점 → LP 개선이 우선"

데이터 소스: 벤치마크 계정의 광고에서 LP URL 추출 가능 (ad_insights_classified에 이미 있음)
