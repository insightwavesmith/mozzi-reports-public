🏥 건강검진 비유

채혈 (수집) — 매일 Meta에서 광고 데이터를 가져옵니다.
CT 촬영 (L1) — AI가 소재의 구조·요소·스타일을 분류합니다.
뇌파 검사 (L2) — 사람 눈이 어디로 가는지 뇌과학 모델로 예측합니다.
혈액 검사표 (L3) — "이 훅은 상위 20%, CTA는 하위 40%" 기준과 비교합니다.
종합 소견서 (L4) — 의사가 "72점, CTA를 바꾸면 90점 가능" 처방전을 씁니다.
생활습관 체크 (LP) — 아무리 건강해도 LP가 나쁘면 전환 안 됨.

계산 방식 상세

ROAS 계산

ROAS (Return On Ad Spend) = 광고비 1원당 매출. ROAS 5 = 1만원 쓰면 5만원 벌어들임.

❌ 기존 (잘못된 방식)

일별 ROAS 단순 평균

예시 — 광고 A의 3일:
• 3/7: spend=11원, 매출=49,000원 → ROAS=4,454 😱
• 3/8: spend=7,792원, 매출=49,000원 → ROAS=6.3
• 3/9: spend=2,092원, 매출=0원 → ROAS=0

단순 평균: (4,454 + 6.3 + 0) ÷ 3 = 1,487 ← 비현실적!

✅ 수정 후 (올바른 방식)

가중 평균 ROAS = 총 매출 ÷ 총 광고비

• 총 매출: 49,000 + 49,000 + 0 = 98,000원
• 총 광고비: 11 + 7,792 + 2,092 = 9,895원

가중 평균: 98,000 ÷ 9,895 = 9.9 ← 현실적!

추가: 총 spend < 10,000원인 광고는 벤치마크에서 제외 (우연 방지)

벤치마크 계산

"이 요소를 가진 소재들의 평균 성과"

예: hook_type = "problem" (문제 제기형):
① L1에서 hook="problem"인 소재 116건
② 각 소재의 daily_ad_insights와 JOIN
③ 가중 평균 ROAS = SUM(전체 매출) ÷ SUM(전체 광고비)
④ P75 ROAS = 상위 25% 소재의 ROAS (이상치에 강건)
→ 9개 축 × 각 값 = 30개 조합

LP 일관성 계산

코사인 유사도 = 두 벡터가 같은 방향인지 (0~1)

① 광고 이미지 → 임베딩 (1536차원 벡터)
② LP 스크린샷 → 임베딩 (1536차원 벡터)
③ cos(광고, LP) = visual_score
+ semantic_score (텍스트↔텍스트)
+ cross_vt, cross_tv (이미지↔텍스트 크로스모달)
→ total_score = 가중 종합

L4 종합 점수 계산

Gemini AI가 모든 데이터를 보고 0~100점 채점

입력: L1 분석 JSON + 실제 ROAS/CTR + L3 벤치마크 + LP 일관성
출력: 5개 영역 점수 → 가중 평균 = overall_score
+ 개선 제안 2~5개 (priority + 구체적 수치)

🚪 비유: 광고 = 가게 간판, LP = 가게 내부. 간판이 아무리 예뻐도 가게 안이 전혀 다른 분위기면 손님은 돌아갑니다. LP 분석은 "간판과 가게 내부가 일치하는지" 체크합니다.

LP 일관성 평가 — 광고와 LP가 같은 이야기를 하나?

방식광고 소재 임베딩 ↔ LP 임베딩 → 코사인 유사도
점수 항목 • visual_score — 광고 이미지 ↔ LP 스크린샷 시각 유사도
• semantic_score — 광고 텍스트 ↔ LP 텍스트 의미 유사도
• cross_vt_score — 광고 이미지 → LP 텍스트 (크로스모달)
• cross_tv_score — LP 이미지 → 광고 텍스트 (크로스모달)
• total_score — 가중 종합
저장creative_lp_consistency
왜 중요한가?
점수 높음 = 광고에서 기대한 것이 LP에서도 보임 → 전환율↑
점수 낮음 = "광고에선 할인 강조했는데 LP에선 안 보임" → 이탈률↑

경쟁사 분석

🆚 비유: 경쟁사 가게 외관·인테리어는 볼 수 있지만, 매출 장부는 절대 못 봅니다. "어떻게 꾸몄는지(구조)"는 비교하되, "잘 되는지(성과)"는 우리 데이터로만 판단합니다.
⚠️ 핵심 한계: 경쟁사 성과 데이터 없음
Meta Ad Library에서 볼 수 있는 건 "어떤 광고를 돌리고 있는지"뿐. ROAS/CTR/매출 같은 성과는 광고주 본인만 볼 수 있어서 좋고 나쁨 판단 불가.

따라서 경쟁사 분석 = "뭘 하고 있는지" 관찰 + "우리와 뭐가 다른지" 구조 비교까지만.

현재 구현

데이터 소스Meta Ad Library API — 브랜드명/키워드로 경쟁사 활성 광고 검색
캐싱competitor_ad_cache — 8,613건 저장
검색 기능브랜드명, 키워드, 기간 필터 → 광고 목록 + 소재 미리보기
분석❌ 현재 없음 — "이런 광고를 돌리고 있다" 확인만 가능

경쟁사에 적용 가능/불가능

분석가능 여부설명
L1 요소 태깅✅ 가능소재 이미지만 있으면 됨 — 훅/스타일/CTA/색감 구조 분석
L2 시선 예측✅ 가능이미지 기반 — 성과 데이터 불필요
L3 벤치마크❌ 불가ROAS/CTR 없이 요소별 성과 통계 계산 불가
L4 종합 점수⚠️ 제한적성과 기반 채점 불가, 구조적 평가만 가능
LP 크롤링✅ 가능경쟁사 LP 방문해서 구조·CTA·헤드라인 분석
게재 기간 추정✅ 가능오래 돌리는 광고 = 효과 있을 가능성 (간접 지표)

수강생 경험 시나리오

Before: "경쟁사가 이런 광고 하고 있대요" → 보고 끝

After:
① 경쟁사 브랜드 검색 → 광고 목록
② 각 광고 L1 분석 → "benefit 훅, professional 스타일, CTA 하단 배치"
③ 내 소재 구조와 비교 → "나는 hook=none, CTA 없음"
④ 게재 기간 긴 광고 참고 → "이 구조가 오래 돌아가고 있으니 효과 있을 것"
⑤ 개선점 도출 → "benefit 훅 추가 + CTA 버튼 추가"

※ 단, "경쟁사 ROAS가 몇이다"는 알 수 없음. 구조적 인사이트만 제공
