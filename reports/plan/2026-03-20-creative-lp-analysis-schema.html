🤖 Gemini 2.5 Pro 프롬프트 설계

소재 분석 프롬프트 (이미지)

You are a senior creative strategist analyzing Meta ad creatives.
Analyze this ad image and return structured JSON.

Evaluate these dimensions:

1. ATTENTION & GAZE
   - attention_score (1-10): How quickly does this stop a scroll?
   - focal_point: What does the eye land on first?
   - face_count, face_emotion, face_prominence

2. MESSAGE & HOOK
   - message_clarity (1-10): How clear is the core message at a glance?
   - primary_message: One sentence summary
   - hook_text: The first copy the eye lands on
   - hook_type: question | bold_claim | pain_point | curiosity_gap | social_proof | before_after | demonstration
   - emotion_appeal: fear | aspiration | social_proof | urgency | curiosity | humor | trust | belonging | exclusivity

3. CTA
   - cta_strength (1-10)
   - cta_text, cta_visible, cta_contrast (1-10), cta_position

4. VISUAL DESIGN
   - dominant_colors (top 5), color_harmony (1-10)
   - text_density: low | medium | high
   - text_readability (1-10), mobile_optimized

5. OVERALL
   - scale_score (1-10): Likelihood of performing well at scale
   - strengths (3), weaknesses (3), recommendations (3)
   - target_audience: Inferred from visuals and copy

Return valid JSON only. No markdown wrapping.

LP 분석 프롬프트 (소재+LP 스크린샷 동시 입력)

You are a conversion optimization expert.
You will receive two images:
1. An ad creative (the ad that was clicked)
2. A landing page screenshot (where the user arrives)

Also provided: LP headline, price, and text content.

Analyze the LANDING PAGE and its CONSISTENCY with the ad:

1. AD↔LP CONGRUENCY (most important)
   - congruency_score (1-10): How well does the LP deliver on the ad's promise?
   - message_match: Does the LP reinforce the ad's core message?
   - visual_match: Is the visual tone consistent?
   - offer_match: Is the price/offer the same?
   - gap_description: Describe any disconnect

2. CONVERSION STRUCTURE
   - conversion_structure: problem_solution_proof_cta | hero_features_social_cta | story_offer_urgency | minimal_direct
   - structure_score (1-10)
   - sections: List page sections in order
   
3. PRICING & OFFER
   - price_visible, price_value, discount_present, urgency_elements

4. TRUST ELEMENTS
   - trust_score (1-10)
   - reviews_present, review_count, social_proof, guarantee

5. LP OVERALL
   - lp_score (1-10)
   - strengths (3), weaknesses (3), recommendations (3)

Return valid JSON only.

📋 Enum 값 정리

hook_type (7종)

question — 질문형 bold_claim — 강한 주장 pain_point — 고통 포인트 curiosity_gap — 호기심 유발 social_proof — 사회적 증거 before_after — 전후 비교 demonstration — 시연

emotion_appeal (9종)

fear — 불안/공포 aspiration — 열망 social_proof — 따라하기 urgency — 긴급성 curiosity — 호기심 humor — 유머 trust — 신뢰 belonging — 소속감 exclusivity — 희소성

conversion_structure (4종)

problem_solution_proof_cta — 문제→해결→증거→CTA hero_features_social_cta — 히어로→기능→사회증거→CTA story_offer_urgency — 스토리→오퍼→긴급성 minimal_direct — 미니멀 직접 전환

🔬 프레임워크 ≠ 정답. 태깅 도구 + ROAS 검증.

⚠️ 핵심 원칙: 위의 분석 축은 "정답"이 아니라 "태깅 도구"입니다. 해외 프레임워크(AdFire, Termo)가 말하는 "좋은 소재"가 한국 시장에서도 좋은 소재인지는 실제 ROAS 데이터로 검증해야 합니다.

왜 해외 프레임워크를 그대로 쓰면 안 되나

한국 이커머스 LP/소재의 현실:

🇰🇷
한국형 LP = 초장문 스크롤
해외: problem→solution→proof→CTA (깔끔한 구조)
한국: 가격+리뷰+상세이미지+체험단+성분표+인증서 폭포
  • 카페24/스마트스토어 상세페이지가 LP의 대부분
  • "스크롤 깊이=깊을수록 나쁘다"가 아님
📦
한국형 CTA ≠ "Shop Now"
"무료배송" "당일발송" "리뷰 12,384개"가 CTA보다 강력
  • 리뷰 수량 = 신뢰의 핵심 지표
  • 인스타/블로그 체험단 = social proof
  • 네이버 스마트스토어 찜 수 = 구매 신호
🎯
한국형 훅 ≠ Pain Point
해외: "고통 → 해결" 구조가 왕
한국: 가격 직구, 리뷰 수, 무료체험이 더 먹힘
  • price_direct — "9,900원부터"
  • review_count — "후기 3만개"
  • free_trial — "7일 무료체험"

Phase 2 — ROAS 검증 (핵심)

3,096건 태깅 완료 후, 실제 ROAS 데이터와 교차 분석합니다.

검증 쿼리 예시

-- ROAS 상위 20% vs 하위 20%의 hook_type 분포 비교
WITH ranked AS (
  SELECT ca.*, d.roas,
    NTILE(5) OVER (ORDER BY d.roas DESC) as roas_quintile
  FROM creative_analysis ca
  JOIN daily_ad_insights d ON ca.ad_id = d.ad_id
  WHERE d.spend > 10000  -- 최소 지출 필터
)
SELECT 
  hook_type,
  COUNT(*) FILTER (WHERE roas_quintile = 1) as top_20pct,
  COUNT(*) FILTER (WHERE roas_quintile = 5) as bottom_20pct,
  AVG(roas) FILTER (WHERE roas_quintile = 1) as top_avg_roas
FROM ranked
GROUP BY hook_type
ORDER BY top_20pct DESC;

발견할 수 있는 패턴 (가설)

발견해외 프레임워크 예측한국 실제 (ROAS 검증)
hook_typepain_point가 최고price_direct, review_count가 더 높을 수 있음
text_densitylow가 좋다 (Meta 권장)high가 ROAS 높을 수 있음 (정보량=신뢰)
face_count얼굴 있으면 좋다제품 클로즈업이 더 좋을 수 있음
CTA"Shop Now" 강조"무료배송" "리뷰 N만개"가 더 효과적일 수 있음
emotionaspiration 최고trust + urgency 조합이 더 좋을 수 있음
congruency높을수록 좋다이건 한국에서도 맞을 가능성 높음 (보편 원칙)

* 위는 가설일 뿐. Phase 2에서 데이터가 답을 줍니다.
