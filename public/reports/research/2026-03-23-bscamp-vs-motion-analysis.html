1. 전체 아키텍처 비교

🟢 bscamp
수집 → 저장 → 분석:광고 → 분석:LP → 순환학습
  • 수집 — Meta Marketing API (collect-daily) → Supabase
  • 저장 — 3계층 (member / benchmark / competitor)
  • 분석 광고 — L1(요소태깅) → L2(DeepGaze 시선예측) → L3(벤치마크) → L4(종합점수)
  • 분석 LP — 눈(시선) → 뇌(Gemini 구조분석) → 탐색(Mixpanel 스크롤) → 결정(Mixpanel 클릭)
  • 순환학습 — 변화감지 → 요소diff → 성과변화 → 데이터화
🟣 Motion
Meta OAuth → AI 태깅 → 리포트 → Runneth AI
  • 수집 — Meta Marketing API OAuth 직접 연결
  • AI 태깅 — 4카테고리 (Visual Format / Persona / Messaging Angle / Hook Tactic)
  • 리포트 — Top Creatives / Comparative / Launch Analysis
  • AI 채팅 — Runneth (계정 데이터 + 벤치마크 RAG)
  • 경쟁사 — Inspo (Ad Library 기반 + 20개 Visual Format 분류)

📐 구조 차이 핵심

  • bscamp: 파이프라인이 깊다. 광고 소재 → LP → 행동 데이터까지 수직으로 연결. 분석 깊이 우선.
  • Motion: 파이프라인이 넓다. 많은 광고주 데이터를 가로로 연결해 벤치마크 파워를 만든다. 데이터 양 우선.
  • 두 접근은 경쟁이 아니라 보완. bscamp가 Motion의 분류체계를 흡수하면 깊이 + 넓이 모두 확보.

3. Motion에서 배울 것 — 카카시 포인트 🥷

적의 기술을 복사한다. 거부감 없이 흡수하면 bscamp가 훨씬 강해진다.

4. bscamp만의 강점 — Motion에 없는 것

🏆 이 5가지는 Motion이 돈을 아무리 써도 당장 따라잡기 어렵다. LP 데이터, 행동 데이터, 시선 예측 — 수집 구조 자체가 다르다.
👁️
시선 예측 (DeepGaze + Gemini)
소재 이미지에서 사용자 시선 흐름 예측. "어디를 먼저 보는지"를 데이터로.

Motion에 아예 없는 기능. 광고 소재 최적화에서 가장 과학적인 접근.
Motion 불가 — 기술 구조 차이
📄
LP 4축 분석
눈(시선) → 뇌(구조분석) → 탐색(스크롤) → 결정(클릭).

광고 소재만 분석하는 Motion과 달리, 랜딩 페이지까지 연결해서 전환 경로 전체를 본다.
Motion에 없음
🔗
소재-LP 일관성 분석
광고에서 약속한 것과 LP에서 보여주는 것이 일치하는지 자동 검증.

불일치 = 전환율 직격. creative_lp_consistency로 정량화.
Motion에 없음
🔄
순환 학습 (Change Detection)
LP/소재 변경 감지 → 변경 전후 성과 비교 → 자동 데이터화.

"뭘 바꿨더니 ROAS가 올랐다/내렸다"를 사람이 아닌 시스템이 학습.
Motion에 없음
🎓
교육 통합
분석 결과 → 수강생에게 "이렇게 고치세요" 가이드 자동 제공.

Motion은 분석 데이터를 주는 것으로 끝. bscamp는 그 데이터를 교육 피드백으로 변환해서 수강생 역량을 직접 향상시킨다. 가장 큰 비즈니스 차별점.

6. Motion 콘텐츠 분석체계 심화 — 배울 것 디테일

Motion의 핵심 경쟁력은 AI 태깅 분류체계(Taxonomy)다. 55만 개 광고를 4개 카테고리로 자동 분류해서, "어떤 유형의 광고가 잘 되는가"를 데이터로 답한다. 이걸 bscamp에 어떻게 흡수할지 하나씩 뜯어보자.

6-1. AI 태깅 4카테고리 상세

🏷️ Visual Format (시각적 형태) — 20개 분류

광고의 "생김새"를 분류한다. 어떻게 만들었는가.

Creator-led (UGC)사람이 직접 카메라 앞에서 말하는 형태. 가장 높은 사용률.

🪝 Hook Tactic (훅 전략) — 5대 분류

$1M+/월 계정에서 가장 많이 쓰이고 효과 좋은 훅 5가지. Motion 벤치마크 $13억 분석에서 추출.

Confession"나도 처음엔 안 믿었는데..." — 솔직한 고백으로 시작. 신뢰 형성.
Bold Claim"이 크림 하나로 주름이 사라졌습니다" — 강한 주장으로 주목 끌기.
Relatability"피부 고민 있는 사람 여기 모여라" — 공감대 형성으로 시작.
Contrast"이전 vs 이후" — 대비로 변화를 시각화.
Curiosity"아무도 안 알려주는 피부관리 비밀" — 궁금증 유발. 정보 격차 이용.

bscamp 적용: analysis_json.hook_tactic 필드 추가. 첫 3초 + 카피 기반으로 Gemini가 분류. 이후 "어떤 훅이 우리 수강생한테 가장 효과적인지" 벤치마크 가능.

🎯 Messaging Angle (메시징 각도)

같은 제품이라도 "어떤 관점으로" 말하는가. Motion은 이걸 자동 분류해서 "어떤 앵글이 ROAS 높은지" 비교한다.

  • Price/Value — 가격, 할인, 가성비 강조
  • Quality/Premium — 품질, 성분, 프리미엄 강조
  • Social Proof — 리뷰 수, 평점, "N만 명이 선택"
  • Emotional — 감성, 스토리텔링, 라이프스타일
  • Functional — 기능, 성능, 스펙 중심
  • Problem→Solution — 고민 제시 → 해결책 제안
  • Urgency/Scarcity — 한정 수량, 타이머, FOMO

bscamp 적용: analysis_json.messaging_angle 필드. 광고 카피 + 첫 프레임 분석으로 분류. "우리 수강생들은 가격 앵글에 치우쳐 있다" → "감성 앵글 테스트해보세요" 처방 가능.

👤 Intended Audience (타겟 추정)

소재가 "누구한테" 말하고 있는지 자동 추정. 연령대, 성별, 관심사.

  • 소재 비주얼 + 카피에서 타겟 자동 추정
  • "이 소재는 20대 여성 / 피부관리 관심층을 타겟"
  • 실제 Meta 타겟팅과 비교 → 불일치 감지

bscamp 적용: analysis_json.intended_audience 필드. Meta API에서 가져온 실제 adset 타겟팅과 비교해서 "소재 톤이 타겟과 안 맞다" 피드백 가능.

6-2. Andromeda 호환 신호 — Motion이 강조하는 핵심

🤖 Andromeda = Meta의 광고 배포 AI

Motion Bootcamp Week 1의 핵심 메시지: "Andromeda는 오가닉 피드에 자연스러운 광고를 더 많이 노출시킨다."

  • 첫 3초가 전부 — 첫 3초가 비슷하면 나머지 달라도 같은 광고로 취급 (Meta 공식)
  • 피드 네이티브 — 유저의 오가닉 피드 미학과 비슷할수록 노출 ↑
  • 시각적 훅 > 카피 — Andromeda는 비주얼을 먼저 평가
  • 과제: 내 타겟 오가닉 피드 스크린샷 캡처 → 내 광고 첫 프레임과 비교 → 유사도 판단

bscamp 적용: 이미 구현됨!

  • Andromeda 유사도 계산: compute-andromeda-similarity.mjs — DB 496건 존재
  • DeepGaze 시선 예측이 "시각적 훅 효과"를 객관적으로 측정
  • 추가할 것: 오가닉 피드 레퍼런스 이미지와 소재 첫 프레임 임베딩 유사도 계산 → "피드 네이티브 점수" 신규 지표

6-3. Comparative Reports — 포맷별 성과 비교

📊 Motion의 킬러 기능: "어떤 유형이 이기고 있는가?"

개별 광고가 아니라, 태그별 그룹으로 묶어서 성과를 비교하는 리포트.

  • "UGC vs 모션그래픽 — 어느 쪽 ROAS가 높아?" → 데이터로 답
  • "Confession 훅 vs Bold Claim 훅 — 어느 쪽이 CTR 높아?" → 데이터로 답
  • "가격 앵글 vs 감성 앵글 — 어느 쪽이 구매까지 가?" → 데이터로 답

bscamp에 없는 것. 개별 소재 분석은 되는데, "유형별 비교"가 안 됨.

구현 방법:

  1. 위 4카테고리 태깅 추가 (Visual Format, Hook Tactic, Messaging Angle, Audience)
  2. 태그별 성과 집계 뷰: SELECT visual_format, AVG(ctr), AVG(roas), COUNT(*) FROM ... GROUP BY visual_format
  3. 프론트에 "비교 분석" 탭 추가 — 두 그룹 선택해서 나란히 비교

예상 효과: 수강생이 "UGC만 만들고 있었는데, 데이터 보니까 Product Demo가 ROAS 2배 높네요" → 바로 전략 전환. 지금까지 감으로 하던 걸 데이터로.

6-4. Naming Convention — 자동 분석의 기반

📛 소재 이름 = 자동 분류의 핵심

Motion은 광고 이름 규칙(Naming Convention)으로 소재를 자동 분류한다.

예: UGC_Confession_PriceFocus_Creator-Kim_V2 → 자동으로:

  • Visual Format = UGC
  • Hook Tactic = Confession
  • Messaging Angle = Price
  • Creator = Kim
  • Version = V2

bscamp 상황: 수강생들은 광고 이름을 체계적으로 안 지음. "봄세일_0320" 같은 식.

bscamp 적용 방법 (2단계):

  1. 지금: Gemini AI가 소재 비주얼+카피 보고 자동 분류 (이름 안 봄) → 정확도 80%+
  2. 나중: 수강생한테 네이밍 컨벤션 교육 → 이름 기반 자동 태깅 → 정확도 100%

→ 단계 1만으로도 충분. AI 태깅이 네이밍보다 더 유연하고 수강생한테 부담이 없다.

6-5. Launch Analysis — 신규 소재 성과 추적

🚀 새 광고 올리면 첫 7일이 승부

Motion은 소재 등록 시점 기준으로 D+1, D+3, D+7, D+14 성과를 자동 추적해서 "이거 위너야 루저야" 빠르게 판단.

  • 위너 기준: 벤치마크 CTR/ROAS 상위 20% 진입
  • 루저 기준: D+3까지 벤치마크 하위 30% → "이거 끄세요" 알림
  • 보류: 중간 → "D+7까지 지켜보세요"

bscamp 적용:

  • creative_media.first_served_at 컬럼 이미 있음 (collect-daily에서 최초 집행일 기록)
  • D+1/3/7/14 성과 집계 뷰 추가 → 프론트에 "신규 소재 추적" 카드
  • 벤치마크 대비 위너/루저 자동 판정 → 슬랙/카톡 알림

예상 효과: 수강생이 "어제 올린 광고 어떻게 됐어요?" → "D+1 CTR 2.3%, 벤치마크 상위 15%입니다. 위너 가능성 높으니 예산 올리세요."

6-6. Runneth AI — 소재 분석 대화형 인터페이스

🤖 "내 광고 왜 안 돼?" → AI가 데이터 기반 답변

Motion의 Runneth: 계정 전체 데이터 + 벤치마크를 RAG로 물고, 자연어로 대화.

  • "지난달 가장 효과 좋았던 훅 유형은?" → 데이터에서 즉시 답변
  • "내 경쟁사 A랑 비교해줘" → Inspo 데이터 + 내 계정 비교
  • "다음주 만들 광고 아이디어 5개" → 성과 데이터 기반 제안

bscamp 적용:

  • qa봇에 account_id 기반 L1~L4 분석 데이터 + 벤치마크 컨텍스트 추가
  • "내 광고 분석해줘" → 해당 수강생의 소재 전체 분석 + 벤치마크 비교 + 처방
  • bscamp 차별점: Runneth는 분석으로 끝. bscamp qa봇은 분석 + 교육 피드백까지. "이 광고의 Hook이 약한 이유는 ~ 이고, 수업 3주차에서 배운 ~ 기법을 적용하면 ~"
