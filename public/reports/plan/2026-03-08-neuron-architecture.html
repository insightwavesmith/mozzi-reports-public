1 핵심 테제

에이전트를 직군(마케터/기획자/개발자)으로 나누는 게 아니라, 하나의 존재가 인지 모드에 따라 다른 뇌를 쓰는 구조.
ADHD 패턴 대화 — 기획 → 잡담 → cron 수리 → 다시 기획. 하나의 컨텍스트 윈도우에 전부 섞이면서 기획 맥락이 잡담에 묻혀 사라진다.
인지 모드(분석/전략/실행/잡담)별로 뇌를 분리. 각 뇌는 자기 도메인 대화만 축적. 잡담이 기획을 오염시키지 않는다.

기존 에이전트와 완전 분리

프로덕션 에이전트는 절대 안 건드린다.

모찌(main), 텐동(tendon), cron-worker = 그대로 유지.
뉴런 아키텍처는 완전히 새로운 에이전트 세트로 빌드 → 테스트 → 검증 후 교체.
현재 프로덕션 (유지) ├─ main (모찌 🍡) → slack:mozzi ├─ tendon (텐동 🍤) → slack:dev-lead └─ cron-worker → cron 전용 새 뉴런 에이전트 (별도 빌드) ├─ neuron-router → 💭 라우터 + 산만 뇌 ├─ neuron-analyst → 🔍 분석 뇌 ├─ neuron-strategist → 🎨 전략 뇌 ├─ neuron-executor → ⚡ 실행 뇌 └─ cron 확장 → 📦 정리 뇌

3 Semantic Router — 뇌 분기 엔진

라우터 = Qwen 2.5 14B (로컬 LLM)

라우터는 BGE-M3(임베딩)가 아니라 Qwen 2.5 14B(로컬 LLM)가 담당한다. 뉘앙스 이해 + 직전 대화 참조가 가능하기 때문.
비교BGE-M3 (임베딩)Qwen 2.5 14B (LLM)
원리cosine similarity언어 이해 + 분류
뉘앙스❌ 애매한 문장 헷갈림✅ 맥락 파악 가능
직전 대화❌ 고려 불가✅ 최근 2~3턴 참조
속도~50ms~0.5~1초
비용무료 (로컬)무료 (로컬)

BGE-M3 = 정리 뇌(메모리 검색). Qwen = 라우터(뇌 분류). 각자 잘하는 걸 시킨다.

라우팅 플로우

Smith: "경쟁사 분석기 기획 정리하자" ↓ neuron-router 메시지 수신 ↓ Qwen에 분류 요청 (Ollama, ~0.5초) 시스템: "분류해: 산만/분석/전략/실행/정리" 직전 대화: [최근 2턴] 메시지: "경쟁사 분석기 기획 정리하자" ↓ Qwen: "전략" ↓ sessions_spawn(agentId="neuron-strategist") → 전략 뇌 활성화 (Opus 4.6) → 응답 → Slack DM 전달

BGE-M3 검증 결과 (참고)

임베딩 유사도 방식도 실제 테스트함. 10개 중 9개 정확(90%). 라우터로는 Qwen을 채택했으나, 정리/recall에 계속 사용.

"ㅋㅋ 오케이" → 💭 산만 (0.820) ✅  |  "점심 뭐먹지" → 💭 산만 (0.725) ✅
"데이터 비교해봐" → 🔍 분석 (0.822) ✅  |  "왜 필요한거야?" → 🎨 전략 (0.698) ✅
"task 만들어줘" → ⚡ 실행 (0.798) ✅  |  "정리해줘" → 📦 정리 (0.786) ✅
"기획 정리하자" → 🔍 분석 (0.662) ⚠️ 전략(0.653)과 근소차 — Qwen이면 해결

라우터가 터지지 않는 이유

라우터가 모든 대화를 컨텍스트에 넣으면 원래 문제(컨텍스트 오염)와 동일.
라우터는 대화를 보관하지 않는다. 현재 메시지 1개 + 직전 2턴만 보고 분류 → 뇌 세션으로 넘기고 잊는다.
라우터 컨텍스트에 남는 것: 현재 메시지 1개 + 직전 2턴 + 라우팅 로그 → 극소량, 안 터짐 라우터 컨텍스트에 안 남는 것: 전략 뇌 30턴 기획 논의 / 분석 뇌 리서치 / 실행 뇌 TASK → 각 뇌 세션에만 존재

4 메모리 아키텍처

인간 뇌와 같은 구조: 워킹 메모리(대화)는 뇌별 분리, 장기 기억(파일)은 전 뇌 공유.
공유 메모리 (장기 기억 — 전 뇌 접근) ├─ MEMORY.md ← 시맨틱 기억 ├─ memory/*.md ← 에피소드 기억 ├─ SOUL.md / USER.md ← 성격/사용자 정보 └─ BGE-M3 벡터 인덱스 ← recall 엔진 뇌별 전용 메모리 (워킹 메모리 — 세션 내) ├─ 전략 뇌 세션 ← 기획 대화만. 잡담 없음. ├─ 분석 뇌 세션 ← 리서치 축적. ├─ 실행 뇌 세션 ← 코드베이스 맥락. └─ 산만 뇌 ← 최근 N턴. 가벼움.

뇌 간 통신 — 핸드오프 파일 (백질)

분석 뇌에서 30턴 대화 후 "task 만들자" → 실행 뇌가 분석 결과를 모른다.
뇌 간 통신은 컨텍스트가 아니라 파일(백질)로 한다. 나가는 뇌가 요약을 쓰고, 들어오는 뇌가 읽는다.
분석 뇌: 30턴 대화, 결론 도출 → memory/handoff.md 기록 "분석 결과: A는 30일+ 광고 12개, 슬라이드 60%" Smith: "task 만들자" Qwen 라우터: → 실행 뇌로 라우팅 실행 뇌: handoff.md 읽고 시작 → TASK.md 작성 (분석 맥락 보존)
3가지 안전장치:

① 핸드오프 파일 — 뇌 전환 시 나가는 뇌가 요약 기록
② 공유 장기 기억 — MEMORY.md는 전 뇌 공유. 중요 결정은 즉시 기록
③ sessions_history 보험 — 최악의 경우 라우터가 이전 뇌 대화 히스토리 조회

정리 뇌 (해마) — 자동 기억 정리

매일 22:00 (수면 = 기억 정리) ① BGE-M3 → 오늘 daily log 임베딩 ② MEMORY.md와 유사도 비교 새 패턴 → 승격 / 중복 → 전략적 망각 ③ Sonnet → 정리 + 구조화 ④ 자동 커밋
