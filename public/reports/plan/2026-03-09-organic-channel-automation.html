2 배포 파이프라인 설계

[bscamp 어드민] ↓ 글 작성/편집 완료 [배포 설정 패널] ↓ 채널 선택 + 발행 트리거 [AI 포맷 변환 API] <— Anthropic API ↓ 채널별 변환본 생성 [channel_distributions 테이블에 큐잉] ↓ +--------------------------------------+ | 병렬 발행 | | +-- 네이버 블로그 -> writePost.json | | +-- 네이버 카페 -> 카페 글쓰기 API | | +-- 구글 SEO -> /insight/[slug] | | +-- GEO -> FAQ schema 마크업 | +--------------------------------------+ ↓ [발행 결과 업데이트 + 어드민 알림]

발행 순서 (권장)

  1. 구글 SEO 서브도메인 — bscamp 자체 서버라 즉시, 실패 없음
  2. 네이버 블로그 — OAuth 토큰만 있으면 안정적
  3. 네이버 카페 — 카페 운영자 토큰 필요
  4. GEO 마크업 — 구글 SEO와 동시 처리 (같은 페이지에 schema 추가)
  5. 뉴스레터 — 기존 흐름 유지 (별도 발행 or 연계)

3 채널별 AI 포맷 변환 규칙

공통 입력 (원본 글)

{ "title": "...", "body": "...(마크다운)", "tags": ["메타광고", "자사몰", ...], "category": "...", "author": "Smith" }
📝
3-1. 네이버 블로그용
네이버 SEO 최적화 + 블로그 친화적 가독성
어조: 친근하지만 전문적 ("~입니다", "~해요" 혼용) 길이: 1500~2500자 구조: 서론(문제제기) → 본론(소제목 3~5개) → 결론(핵심 요약) 소제목: ✓ 이모지 활용, 굵게 처리 키워드: 제목 + 첫 문단 + 소제목에 3회 이상 태그: 10개 (광고 관련 + 브랜드 관련 혼합) CTA: 마지막에 "자사몰사관학교 2기 →" 링크 멘션 금지: 과도한 줄바꿈, 광고성 표현 출력: { title, content(HTML허용), tags[], category }
☕
3-2. 네이버 카페용
커뮤니티 참여 유도 + 정보 공유
어조: 구어체 ("여러분", "공유드려요") 길이: 800~1200자 (카페는 짧게 핵심만) 구조: 인트로(왜 공유하는지) → 핵심 3가지 → 마무리 질문 마무리: 댓글 유도 질문 필수 ("여러분은 어떻게 운영하세요?") 카테고리: 정보공유 or 마케팅팁 금지: 외부 링크 과도 삽입 (카페 스팸 필터) 출력: { subject, content(HTML), categoryId }
🔍
3-3. 구글 SEO 서브도메인용
구글 크롤링 최적화 + 롱테일 키워드 포지셔닝
어조: 명확하고 정보 중심적 ("~입니다", 3인칭 혼용) 길이: 1200~2000자 (품질 > 양) 구조: H1(제목) → H2(소제목) → H3(세부) 계층 명확 메타 디스크립션: 160자 이내 별도 생성 키워드 밀도: 1~2% (과도한 반복 금지) 내부 링크: bscamp.vercel.app 링크 1~2개 구조화 데이터: Article schema 포함 출력: { slug, title, metaDescription, ogTitle, ogDescription, body(md), structuredData }
🤖
3-4. GEO (AI 검색) 마크업용
ChatGPT / Perplexity / Google AI Overview 답변 소스 채택
Q&A 쌍 3~5개 추출 (글의 핵심 질문-답변) 질문: 사용자가 실제 검색할 법한 자연어 답변: 40~120자 명확한 답변 (너무 길면 AI가 발췌 안 함) HowTo schema: 단계별 절차 있을 경우 생성 Speakable: 음성 검색 대비 출처 표시: "자사몰사관학교 bscamp" 명시 출력 예시: { "faqSchema": { "@context": "https://schema.org", "@type": "FAQPage", "mainEntity": [{ "@type": "Question", "name": "메타 광고 ROAS가 낮을 때 어떻게 해야 하나요?", "acceptedAnswer": { "@type": "Answer", "text": "..." } }] }, "speakable": ["첫 번째 답변 텍스트", ...] }

4 DB 스키마 추가

-- channel_distributions CREATE TABLE channel_distributions ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, content_id UUID REFERENCES contents(id) ON DELETE CASCADE, channel_type TEXT NOT NULL, -- 'naver_blog' | 'naver_cafe' | 'google_seo' | 'geo' | 'newsletter' status TEXT DEFAULT 'pending', -- 'pending' | 'transforming' | 'ready' | 'published' | 'failed' transformed_content JSONB, -- 채널별 변환된 컨텐츠 external_url TEXT, -- 발행된 외부 URL external_id TEXT, -- 네이버 postId 등 error_message TEXT, scheduled_at TIMESTAMPTZ, -- 예약 발행 published_at TIMESTAMPTZ, created_at TIMESTAMPTZ DEFAULT NOW(), updated_at TIMESTAMPTZ DEFAULT NOW() ); -- channel_credentials CREATE TABLE channel_credentials ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, channel_type TEXT UNIQUE NOT NULL, credentials JSONB NOT NULL, -- 암호화 저장 (access_token, refresh_token 등) expires_at TIMESTAMPTZ, created_at TIMESTAMPTZ DEFAULT NOW(), updated_at TIMESTAMPTZ DEFAULT NOW() );
⚠ 기존 service_secrets 테이블 활용 가능 — channel_type별 row 추가로 대체 검토

7 구글 SEO 서브도메인 구조

bscamp 내 공개 접근 가능한 인사이트 페이지 생성. 비로그인도 접근 가능해야 구글 크롤링이 된다.

경로설명
/insight목록 페이지 (SEO 최적화)
/insight/[slug]개별 아티클 (구조화 데이터 포함)
  • 공개 접근: 비로그인도 접근 가능 (구글 크롤링 필수)
  • sitemap.xml 자동 업데이트: 발행 시 sitemap에 추가
  • robots.txt: /insight/* Allow
  • 내부 CTA: "더 배우고 싶다면 → 자사몰사관학교 2기"

9 네이버 API 인증 참고

필요한 것

  1. Naver Developers에서 앱 등록
  2. 권한: blog (블로그 읽기/쓰기), cafe (카페 가입, 글쓰기)
  3. 콜백 URL: https://bscamp.vercel.app/api/admin/naver/callback
  4. Client ID + Secret → 환경변수

환경변수 추가 (.env.local + Vercel)

NAVER_CLIENT_ID=... NAVER_CLIENT_SECRET=... NAVER_CAFE_CLUB_ID=... # 운영 카페 ID

토큰 저장 (service_secrets 테이블)

key: 'naver_blog_credentials' value: { access_token, refresh_token, expires_at }
