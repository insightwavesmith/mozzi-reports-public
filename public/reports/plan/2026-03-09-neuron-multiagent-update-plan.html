P0

🔗 체인 자동 연결 실패 — 라우터가 다음 뇌 spawn 누락

오늘 실제 발생: analyst 결과 받고 strategist로 체인해야 했는데 누락. 체인 판단이 프롬프트 규칙에만 의존 → 컨텍스트 길어지면 잊어버림.

현재 문제

  • 체인 여부를 라우터 판단에만 의존
  • 명시적 신호 없음 → 컨텍스트 길면 누락
  • 뇌 결과물에 "다음 단계" 구조화 없음

해결 방향: CHAIN 태그

  • 각 뇌 결과물에 [CHAIN:뇌이름] 태그 강제
  • 라우터: 태그 감지 → 자동 spawn (기억 불필요)
  • analyst/strategist AGENTS.md에 태그 규칙 추가
# 뇌 결과물 마지막 줄에 반드시 포함
task에 체인 암시 (예: "분석 후 전략 수립"):
  응답 마지막: [CHAIN:strategist]

task에 실행 포함 (예: "전략 만들어 실행까지"):
  응답 마지막: [CHAIN:executor]

최종 전달 (체인 없음):
  응답 마지막: [CHAIN:none]

# 라우터 AGENTS.md — 체인 처리 규칙 (신규 추가)
결과 받으면 마지막 줄 CHAIN 태그 확인:
  [CHAIN:strategist] → strategist spawn(결과 + handoff.md 포함)
  [CHAIN:executor]   → executor spawn(결과 + handoff.md 포함)
  [CHAIN:none] 또는 태그 없음 → Smith님에게 전달
핵심 원칙: 라우터의 "기억"에 의존하지 말고, 뇌의 결과물 자체에 체인 신호를 임베드. 태그가 있으면 라우터는 그냥 실행만. 뇌가 자기 다음 단계를 명시.
P0

🧠 뇌 프롬프트가 "에이전트 관점" — 독립 에이전트처럼 작성됨

Smith님 의도: 모찌 한 명의 뇌 일부. 독립 목표·판단이 있는 에이전트가 아니라, 전체 인격(모찌)의 특정 인지 기능. 현재 AGENTS.md들은 이 구분이 없음.

에이전트 관점 (현재)

  • "이 에이전트는 X를 담당한다"
  • 독립적 목표 설정 언어
  • 다른 뇌를 "다른 에이전트"로 인식
  • 자기 영역 외 = "거절"하는 뉘앙스

뇌 관점 (목표)

  • "나는 모찌의 분석 기능이다"
  • 모찌의 전체 목표를 위해 동작
  • 다른 뇌 = 같은 인격의 다른 기능
  • 자기 영역 외 = 다른 기능이 더 잘하므로 협력
핵심 구분: 에이전트는 "나는 무엇을 하겠다"고 결정하는 주체. 뇌는 "모찌가 이 상황을 어떻게 처리하는가"를 수행하는 기능. 주체는 내(뇌)가 아닌 모찌(전체 인격). → 각 AGENTS.md 정체성 섹션 전면 리라이팅 필요.
P0

🚦 업무 시퀀스 미보장 — 동시 업무 혼선

여러 메시지 동시 입력 → 다중 spawn → SESSION-STATE/handoff 동시 write 충돌 가능. 사람처럼 하나의 업무에 집중, 진행 중이면 새 요청 큐잉 또는 알림 필요.

현재 문제

  • 동시 메시지 → 다중 spawn 가능
  • SESSION-STATE.md 동시 write → 충돌
  • handoff.md 동시 append → 혼선
  • 현재 상태(뭐 하고 있는지) 표현 없음

해결: 파일 기반 BUSY 플래그

  • SESSION-STATE.md에 BUSY 섹션 추가
  • spawn 전: BUSY 확인 → BUSY면 알림
  • spawn 시: BUSY:true + 작업명 기록
  • 완료 시: 뇌가 BUSY:false 초기화
## 업무 상태 (SESSION-STATE.md에 추가 — 라우터 spawn 전 반드시 확인)
BUSY: false
CURRENT_TASK: null
TASK_STARTED: null

# spawn 시 라우터가 기록:
BUSY: true
CURRENT_TASK: "bscamp 오가닉 채널 자동화 전략"
TASK_STARTED: "2026-03-09T10:30:00+09:00"

# 완료 시 뇌가 초기화 (AGENTS.md에 추가):
SESSION-STATE.md 업데이트: BUSY: false, CURRENT_TASK: null

🔄 라우터 업무 수락 플로우

  1. 새 메시지 수신 → SESSION-STATE.md BUSY 확인
  2. BUSY: false → 정상 spawn + BUSY:true 기록
  3. BUSY: true → "지금 [작업] 중이야, 끝나면 바로 할게!" 직접 답변
  4. 완료 후 뇌가 BUSY:false 초기화 (AGENTS.md 완료 규칙에 추가)
  5. stale 방지: TASK_STARTED 15분 초과 시 라우터가 자동 BUSY:false
P1

📝 analyst handoff 기록 불완전

analyst AGENTS.md "항상 기록" 규칙은 있으나, 체인 이전 전달 시 다음 뇌가 읽기 어려운 포맷. CHAIN 태그와 handoff 포맷 연동 필요.

현재 상태

  • "항상 기록" 규칙은 있음
  • 체인용/최종용 구분이 task 키워드에 의존
  • handoff에 CHAIN 태그 연동 없음
  • 다음 뇌가 읽을 때 핵심 찾기 어려움

개선 방향

  • CHAIN 태그 있을 때 → 체인용 handoff 포맷 (간결)
  • handoff에 "다음 뇌 읽기용 요약" 필드 추가
  • 핵심 결론 → 근거 → 미결정 → 다음 뇌 지시 순서
  • CHAIN:none 시 → 최종 전달용 (상세)
P1

⚡ executor AGENTS.md handoff 규칙 미반영

executor만 "체인용일 때만" handoff 기록. 다른 뇌들은 모두 "항상 기록"으로 통일됐는데 executor만 예외. 일관성 깨짐.

현재 상태

  • handoff.md 작성: "체인용일 때만"
  • 다른 뇌와 규칙 불일치

개선 방향

  • "항상, 예외 없음"으로 변경
  • 다른 뇌들과 동일한 handoff 포맷 채택
  • 커밋 해시, 변경 파일, 빌드 결과 포함
