1 설계 철학

각 뇌가 자기 영역의 컨텍스트를 깊이 축적하는 것은 돈이 들어도 이득이다.
여러 번 반복해서 설명하는 게 진짜 손해다.

하나의 컨텍스트 윈도우에 잡담/기획/개발/리서치를 다 섞으면, 기획 맥락이 잡담에 묻혀 사라진다. 그러면 매번 "아까 그거 뭐였지?" 반복. 이게 진짜 비용.

해법: 인지 모드별로 뇌를 분리. 전략 뇌는 기획만 30턴 축적. 분석 뇌는 리서치만 축적. 잡담이 끼어들 여지 없음. 뇌를 바꿔도 파일로 인수인계하니까 맥락은 살아있음.

핵심: 하나의 인격(모찌), 여러 개의 뇌. Smith님 입장에서는 모찌 한 명과 대화. 뒤에서 뇌가 바뀌는 건 보이지 않음.

2 인간 뇌 → AI 매핑

MAP 논문(Nature 2025, Microsoft Research)이 전두엽 계획 과정을 6개 모듈로 분해한 것에 기반. 우리 구조는 이 중 핵심 4개를 구현하고, 나머지는 Phase 2에서 추가.

3 3층 아키텍처

Layer 1 — 시상 · OpenClaw Hook
메시지가 들어오면 Sonnet이 분류 (코드 레벨, 3초 timeout)
→ casual / analyst / strategist / executor / memory
→ [ROUTE:X] 태그를 메시지에 주입
→ Supabase 로깅 (fire-and-forget)

왜 코드인가: LLM한테 맡기면 "내가 답할게" 하고 직접 처리함. 코드로 강제해야 100% 결정적.

OpenClaw hooks.internal — message:received 이벤트 인터셉트
↓ [ROUTE:X] 태그 주입됨
Layer 2 — 전두극피질 · Orchestrator
neuron-router (Sonnet 4.6) — 위임하고 판단. 직접 작업 안 함.

① 태그 확인 → 올바른 뇌에 spawn
② SESSION-STATE + handoff를 attachments로 직접 전달
③ 결과 받으면 → 다음 뇌 체인 판단
   (예: analyst 결과 → executor에 넘기기)
④ 최종 결과를 Smith님에게 전달
⑤ casual/memory만 직접 처리

OpenClaw sessions_spawn — agentId + attachments + thinking
OpenClaw subagents.allowAgents — 허용된 뇌 목록
OpenClaw subagents.thinking: "high" — spawn 시 자동 적용
OpenClaw subagents.runTimeoutSeconds: 300 — 5분 제한
↓ sessions_spawn(attachments=[SESSION-STATE, handoff])
Layer 3 — 피질 · 전문 뇌들
각 뇌는 자기 영역의 컨텍스트만 깊이 축적.
결과는 3~5줄 자체 압축 후 반환. 상세는 handoff.md에 기록.

OpenClaw 에이전트별 model + skills + AGENTS.md 독립 설정
OpenClaw model.fallbacks — Opus 실패 시 Sonnet 자동 전환
OpenClaw workspace 분리 — symlink로 메모리만 공유
↓ 결과 압축 반환 + handoff.md 기록
Layer 4 — 신피질 · 공유 메모리
모든 뇌가 symlink로 공유하는 장기 기억.

SESSION-STATE.md — 작업 기억 (WAL)
MEMORY.md — 장기 기억
memory/handoff.md — 뇌 간 인수인계 (백질)
memory/YYYY-MM-DD.md — 에피소드 기억

OpenClaw memorySearch — BGE-M3 로컬 임베딩
OpenClaw compaction.memoryFlush — 컴팩션 전 자동 저장
OpenClaw session.maintenance: enforce — 오래된 세션 자동 정리

5 왜 축적이 이득인가

"각 뇌는 본인의 컨텍스트가 축적되고, 많이 가지고 있어도 돈이 들어도 이득이다. 여러 번 반복하는 게 다 손해다."
반복의 비용 (지금):
• 기획 30턴 → 잡담 끼어듦 → 기획 맥락 증발
• "아까 그거 뭐였지?" → 재설명 5분
• 컴팩션 후 → "뭐 하고 있었죠?" → 파일 읽기 3~4회
• 같은 코드 구조 매번 설명 → 매 세션 10분 낭비

축적의 이득 (v2):
• 전략 뇌: 기획 논의 30턴이 온전히 축적. 5번째 질문에도 1번째 맥락 살아있음
• 분석 뇌: 한번 조사한 건 세션에 남아있음. 재조사 불필요
• 실행 뇌: 코드베이스 구조 한번 파악 → 다음 TASK에서 바로 시작
• 토큰 비용 ↑ but 반복 비용 ↓↓↓ = 순이익

one-shot(mode="run")이지만, handoff.md가 세션 간 기억을 연결한다. 뇌가 바뀌어도 파일에 핵심이 남아있으니 "처음부터 다시" 없음.

7 데이터 흐름

단순 요청

Smith: "경쟁사 데이터 찾아봐" │ Hook Sonnet 분류 → analyst [ROUTE:analyst] 태그 주입 │ 라우터 태그 확인 SESSION-STATE.md 읽기 → sessions_spawn( agentId: "neuron-analyst" attachments: [SESSION-STATE 발췌] ) │ 분석 뇌 리서치 수행 → handoff.md에 상세 기록 → 3~5줄 요약 반환 │ 라우터 → Smith님에게 🔍 결과 전달

복합 요청 (체인)

Smith: "경쟁사 분석하고 TASK 만들어줘" │ Hook → strategist (복합 판단) │ 라우터 "분석 먼저 필요" → analyst spawn │ 분석 뇌 리서치 → 요약 반환 + handoff 기록 │ 라우터 결과 수신 "다음은 TASK → executor" ← 체인 판단 → executor spawn (attachments: 분석 요약) │ 실행 뇌 handoff.md 참조 → TASK.md 작성 → 요약 반환 │ 라우터 → Smith님에게 최종 전달
체인 = v2 핵심. v1은 한 뇌만 spawn하고 끝. v2는 라우터가 결과를 보고 "다음 뇌 필요?" 판단.
