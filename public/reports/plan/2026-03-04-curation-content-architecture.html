1. 현재 문제

핵심 문제: 소스가 flat list로 쌓이기만 해서, "뭐가 좋은 콘텐츠인지" 판단이 전부 관리자 몫. 구조화, 분류, 우선순위가 없음.

관리자의 현재 워크플로우

📥
소스 수집
Notion sync, URL 크롤링 등으로 자동/수동 수집
→
😵
Flat List 스크롤
분류 없이 전부 나열. 좋은 글 찾으려면 하나하나 열어봐야 함
→
✍️
수동 선택+생성
최대 4개 골라서 AI 생성. 발행 후 관리 어려움

3-2. AI 스코어링 구조

축 측정 방법 가중치 단계
관련성 관리자 설정 키워드와 소스 제목/본문 매칭 빈도 50% MVP
신선도 수집일 기준 시간 감쇠 (최근일수록 높음) 30% MVP
출처 신뢰도 관리자가 자주 선택하는 출처 가중치 20% v1.1
참여 예측 발행 후 조회수/좋아요 피드백 학습 보너스 v2.0
MVP 스코어 공식: score = (키워드 매칭 × 0.5) + (신선도 × 0.3) + (기본값 20)
키워드 목록은 관리자가 /admin/settings에서 설정. 예: "메타광고, ROAS, 카페24, 전환율, ASC, CAPI"

5. 구현 로드맵

1
Phase 1 — 안전망 + 인박스 MVP
flat list 탈출. 빠른 체감 효과.
  • 소프트 삭제 + 삭제 시 소스 자동 복구
  • AI 관련성 스코어 컬럼 추가 (키워드 매칭 기반)
  • 우선순위 인박스 뷰 (HIGH/MED/LOW 구간)
  • 3액션 버튼: 글 쓰기 / 나중에 / 건너뜀
  • dismissed 복원 기능
2
Phase 2 — 토픽 + 양방향 연결
구조화. "어떤 주제를 다뤘고 뭐가 비었는지" 한눈에.
  • 토픽 카테고리 관리 (관리자 설정)
  • AI 자동 토픽 태깅 (제목/본문 기반)
  • 토픽 클러스터 그리드 뷰
  • 소스↔생성물 양방향 연결 표시
  • 현재 프롬프트로 재생성 버튼
3
Phase 3 — 피드백 루프
발행 후 데이터가 스코어링을 개선.
  • 발행 후 조회수/좋아요 → 스코어 피드백
  • 관리자 선택 패턴 학습 → 추천 고도화
  • 토픽별 성과 대시보드
  • 벌크 액션 + 고급 검색/정렬
