스코어 축 계산 방법 범위 단계

오늘=100, 어제=90, 이번주=70, 이번달=40, 그 이전=10		 0-100 MVP

인기 토픽이면 +10점 보너스		 +0~20 v2.0

토픽 분류 알고리즘

단계방법정밀도
MVP topic_categories.keywords 매칭 — "ROAS, 광고비, CPM" → 메타광고 70%
v1.1 Sonnet 1-shot 분류 — 제목+요약 보고 토픽 배열 반환 90%
v2.0 임베딩 유사도 — 각 토픽의 대표 벡터와 소스 벡터 코사인 유사도 95%

07 에러 핸들링 & 그레이스풀 폴백

각 레이어에서 실패 시 어떻게 대응하는가

스코어링 실패
→ relevance_score = 50 (중립 기본값)
→ 인박스에서 MED 그룹에 배치
→ 관리자가 수동 판단
토픽 분류 실패
→ topic_tags = ["미분류"]
→ 토픽 뷰에서 "미분류" 그룹에 표시
→ 관리자가 수동 태깅 가능
AI 생성 실패
→ 소스 상태 "reviewing"으로 유지
→ 에러 메시지 표시
→ 재시도 버튼 제공
소프트 삭제 시 source_ref 파싱 실패
→ 생성물만 archived 처리
→ 소스 복구 실패 로그 기록
→ 관리자에게 수동 복구 안내
Notion sync 실패
→ 기존 데이터 유지
→ 다음 크론에서 재시도
→ 3회 연속 실패 시 관리자 알림
중복 소스 수집
→ 제목 유사도 80%+ 시 경고 표시
→ 자동 스킵하지 않음 (관리자 판단)
→ "유사 콘텐츠 있음" 배지
